# 检测区域功能使用说明

## 📖 功能概述

为 LD Radar Monitor 添加了多边形检测区域功能，支持：
- ✅ 最多 8 个自定义检测区域
- ✅ 桌面端和移动端绘制
- ✅ 实时触发状态显示
- ✅ 自动保存到后端
- ✅ 完善的容错处理

---

## 🎯 如何使用

### 1. 绘制新区域

#### 桌面端（鼠标）
1. 点击侧边栏的 **"+ 新建区域"** 按钮
2. 在画布上**点击**添加顶点（至少 3 个）
3. **双击**完成绘制，或**右键**取消
4. 区域自动保存

#### 移动端（触摸）
1. 点击侧边栏的 **"+ 新建区域"** 按钮
2. 在画布上**点击**添加顶点（至少 3 个）
3. 点击底部工具栏的 **"✓ 完成"** 按钮
4. 点击 **"✕ 取消"** 可放弃绘制

### 2. 查看触发状态

- **正常状态**：区域显示为半透明填充 + 彩色边框
- **触发状态**：区域变为红色高亮 + 闪烁动画
- 侧边栏区域列表显示"触发中"标签

### 3. 删除区域

当前版本暂不支持直接删除，需要：
1. 在后端手动清除配置
2. 或等待后续版本更新

## 📊 数据格式

### WebSocket 消息

#### 后端 → 前端：配置消息
```json
{
  "type": "config",
  "zones": [
    {
      "id": 1,
      "name": "门口",
      "points": [[1.5, 3.0], [3.5, 3.0], [3.5, 5.0], [1.5, 5.0]],
      "color": "#ff6b6b",
      "enabled": true
    }
  ]
}
```

#### 后端 → 前端：实时数据
```json
{
  "type": "data",
  "t": [
    [1, 1.5, 2.3, 0.5],
    [2, -0.8, 3.1, 0.2]
  ],
  "z": [1, 1]
}
```

说明：
- `t`: 目标数组，每个元素为 `[id, x, y, speed]`
- `z`: 区域触发状态，每两个元素为一组 `[zoneId, triggered]`

#### 前端 → 后端：保存配置
```json
{
  "type": "save_zones",
  "zones": [...]
}
```

## ⚙️ 技术细节

### ZoneManager 类

**主要方法：**
- `addZone(zoneData)` - 添加区域
- `deleteZone(zoneId)` - 删除区域
- `setZones(zonesArray)` - 批量设置区域
- `getZonesArray()` - 获取所有区域
- `updateTriggerStates(triggerStates)` - 更新触发状态
- `isPointInPolygon(px, py, polygon)` - 点在多边形内判断
- `exportConfig()` - 导出配置（用于保存）

**配置选项：**
```javascript
const zoneManager = new ZoneManager({
  maxZones: 8,           // 最大区域数
  maxPoints: 10,         // 每个区域最大顶点数
  coordRange: 20         // 坐标范围 ±20m
});
```

### ZoneEditor 类

**主要方法：**
- `startDrawing()` - 开始绘制
- `finishDrawing()` - 完成绘制
- `cancelDrawing()` - 取消绘制

**交互模式：**
- 桌面端：鼠标点击 + 双击/右键
- 移动端：触摸点击 + 工具栏按钮

### Canvas 渲染

**新增方法：**
- `setZones(zones)` - 设置区域数据
- `setPreviewPolygon(points)` - 设置预览多边形
- `clearPreview()` - 清除预览
- `_drawZones(ctx)` - 绘制所有区域
- `_drawZone(ctx, zone)` - 绘制单个区域
- `_drawPreviewPolygon(ctx, points)` - 绘制预览

## ⚠️ 注意事项

### 数据验证
- 顶点数量：3-10 个
- 坐标范围：±20 米
- 颜色格式：`#RRGGBB`
- 最多 8 个区域

### 性能优化
- 渲染帧率：≥ 50 FPS（8 区域 + 20 目标）
- WebSocket 频率：10 Hz
- 内存占用：< 50 MB

### 容错处理
- 无效数据被过滤（有日志）
- 网络错误有提示
- Canvas 渲染异常不崩溃
- 连接断开时清空过时数据

## 🐛 常见问题

### Q1: 绘制时没有反应？
**A:** 检查是否点击了"新建区域"按钮进入绘制模式。

### Q2: 区域没有保存？
**A:** 检查网络连接，查看 Console 是否有错误日志。

### Q3: 触发状态不更新？
**A:** 确认 WebSocket 连接正常，检查后端是否推送 `data` 消息。

### Q4: 移动端无法绘制？
**A:** 确保使用现代浏览器，检查触摸事件是否被阻止。
